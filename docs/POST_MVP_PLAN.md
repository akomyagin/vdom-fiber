# POST_MVP_PLAN — vdom-fiber

Расширения после того, как MVP (Этапы 0–6) закрыт и демо-счётчик работает end-to-end
через собственный VDOM + Fiber + scheduler + hooks. Порядок — ориентировочный,
берётся по интересу и сложности.

## 1. `useEffect`

- Хук с массивом зависимостей; эффекты запускаются **после** commit-фазы.
- Cleanup-функция вызывается перед повторным запуском эффекта и при размонтировании.
- Требует: список эффектов на fiber, сравнение deps, фаза «passive effects» после commit.

## 2. Context API

- `createContext` / провайдер / `useContext`.
- Пробрасывание значения вниз по дереву fiber'ов без props-drilling.
- Инвалидация подписчиков при смене значения контекста (с учётом прерываемого рендера).

## 3. Concurrent-фичи сверх базового планировщика

- `startTransition` — помечать обновления как низкоприоритетные (transitions),
  чтобы они уступали user-blocking апдейтам.
- Батчинг нескольких `setState` в один рендер-проход.
- Отмена/перезапуск устаревшего WIP-дерева при поступлении более свежего апдейта
  (гарантия «tearing-free» результата).
- `useTransition` / индикатор pending для демонстрации.

## 4. Полноценный JSX

- В MVP осознанно выбран hyperscript-API без JSX (см. TECHNICAL_PLAN «Решение по JSX»).
- Здесь — включить JSX как отдельный слой: `jsxImportSource` для нового JSX-transform
  или babel-плагин, компилирующий JSX в вызовы `h`/`createElement`.
- Обновить пример-демо на JSX, оставив hyperscript-API рабочим для тех, кто без сборки.

## 5. Прочие nice-to-have

- Дополнительные хуки: `useMemo`, `useCallback`, `useRef`, `useReducer`.
- Фрагменты (`<>…</>`) и порталы.
- Простейшая обработка ошибок рендера (аналог error boundary).
- Дев-логирование/трейсинг единиц работы планировщика для наглядной демонстрации
  прерываний (визуализация «render → yield → resume»).
- Мини-бенчмарк: сравнить синхронный проход и прерываемый на большом дереве.

## Явно вне скоупа

- SSR / гидратация.
- Синтетическая система событий уровня React (делегирование на корень).
- Совместимость с экосистемой React (это самостоятельный учебный движок, не замена React).
