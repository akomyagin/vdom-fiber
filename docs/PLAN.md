# PLAN — vdom-fiber

## Видение

`vdom-fiber` — учебная реализация мини-React с нуля на TypeScript. Цель не в том,
чтобы конкурировать с React, а в том, чтобы своими руками пройти путь от наивного
Virtual DOM до **прерываемого (interruptible) рендеринга с приоритетами**, как в
настоящем React Fiber.

Проект сделан ради искренней технической сложности и обучения: экономика и
пользовательская база не рассматриваются. Ключевая интеллектуальная ценность
сосредоточена не в VDOM-диффинге (это известный учебный путь с готовыми
референсами), а в Fiber-подобном планировщике — возможности прервать обход дерева
на середине, вернуть управление браузеру и продолжить с сохранённого состояния.

## Что строим (в двух словах)

- Свой `createElement` / VNode-дерево (без зависимости от React).
- Синхронный mount виртуального дерева в реальный DOM.
- VDOM-диффинг (reconciliation) для повторных рендеров.
- Fiber — связное (linked-list) дерево единиц работы вместо рекурсии.
- Планировщик с time-slicing, приоритетами и прерыванием через `MessageChannel`.
- Минимальный `useState`, чтобы всё это можно было продемонстрировать вживую.

## Верхнеуровневая архитектура

```
createElement ──▶ VNode-дерево (types.ts)
                      │
      Этап 2          ▼
   render() ──▶ первая синхронная отрисовка в DOM (mount)
                      │
      Этап 3          ▼
   reconcile() ──▶ диффинг prev/next VNode, минимальные мутации DOM
                      │
      Этап 4          ▼
   Fiber-дерево ──▶ child/sibling/return указатели, прерываемый обход
                      │
      Этап 5          ▼
   scheduler ──▶ work loop + shouldYield() + приоритеты + preemption
                      │
      Этап 6          ▼
   useState ──▶ обновление состояния запускает re-render через scheduler
```

Слои держим тонкими и разнесёнными по модулям (`createElement`, `render`,
`reconciler`, `fiber`, `scheduler`, `hooks`), чтобы поздние этапы читались
естественно и тестировались изолированно.

## Этапы (MVP)

| Этап | Название | Суть | Сложность |
|---|---|---|---|
| 0 | Скелет | package.json, tsconfig, Vitest+jsdom, модули-заглушки | тривиально |
| 1 | `createElement` + VNode | фабрика виртуальных узлов, нормализация детей | низкая |
| 2 | Синхронный mount | первая отрисовка VNode-дерева в реальный DOM | низкая-средняя |
| 3 | VDOM-диффинг | reconciliation prev/next для повторных рендеров | средняя |
| 4 | Fiber-структура | linked-list дерево с возможностью приостановки обхода | средняя-высокая |
| 5 | Планировщик | time-slicing, приоритеты, прерывание через `MessageChannel` | **высокая** |
| 6 | Hooks (`useState`) | минимальный state, чтобы демонстрация имела смысл | средняя |

**Порядок принципиален:** Fiber-планировщик (Этапы 4–5) — самая сложная часть и
идёт НЕ первым. Сначала работающий синхронный VDOM (Этапы 1–3), потом переход на
прерываемую модель. Так на каждом этапе есть работающий и тестируемый результат.

Подробная техническая разбивка каждого этапа — в
[TECHNICAL_PLAN.md](./TECHNICAL_PLAN.md).

## Критерии готовности MVP

- Демо-приложение (счётчик на `useState`) монтируется и обновляется через свой рендер.
- Обновления идут через планировщик; длинный рендер прерывается и не блокирует UI.
- Высокоприоритетное обновление вытесняет фоновый рендер.
- Каждый модуль покрыт тестами на итоговое DOM-дерево (jsdom + Vitest).
